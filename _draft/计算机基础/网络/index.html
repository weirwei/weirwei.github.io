TCP、UDP、IP协议分别属于什么层？

  • tcp和udp 协议属于传输层
  • IP协议属于网络层

网络中传输层有什么作用？它有那些协议？

  • 主机不同进程间通信。传输层负责在在主机之间提供可靠的数字传输服务。通过流量控制、分段/重组和差错控制，确保数据在网络中正确无误地传输
  • 常见的协议有：TCP、UDP

简述HTTP1.0、1.1、2.0的区别

  • 1.0：支持多媒体对象
  • 1.1：支持长连接、管道化、缓存处理、断点传输
  • 2.0：性能提升、多路复用、头部压缩、服务端推送

keep-alive 长连接：减少连接建立和释放的成本

多路复用：

  • 二进制分帧是基础，通信单位是帧
  • 多请求并行，不依赖多tcp连接
  • 并行在一个tcp连接交互多种类型信息

头部压缩：

  • 维护一个静态的字典
  • 把内容进行哈夫曼编码进行压缩

服务端推送：

  • 服务端在客户端请求一个资源时，主动推送相关资源。
  • 相关性判断：
    • 静态分析：如html中的css引用
    • 请求上下文：
    • 预加载指令：如，服务器可以在响应中包含Link头字段，指明与当前请求的资源相关的其他资源需要被推送。

请描述常见的HTTP请求方法和使用场景

  • GET：请求某个资源
  • HEAD：和get类似，但是服务器只返回首部。通常用来检查
  • POST：向服务器写入数据
  • TRACE：观察请求报文到达服务器的最终样子
  • PUT：向服务器写入资源
  • DELETE：删除指定资源
  • OPTION：返回资源支持操作的方法

常见的状态码404、503、504分别表示什么意思？怎么定位？

  • 2xx：成功

  • 3xx：重定向

  • 4xx：客户端错误

  • 5xx：服务端错误

  • 304：Not Modified 请求资源未修改，服务端不返回任何资源，客户端可以从本地获取

  • 404：Not Found

  • 502：Bad Gateway

  • 503：Service Unavailable 宕机

  • 504：网关响应超时

MD5是加密算法嘛？为什么？什么是加盐处理

哈希算法：又叫散列算法。把消息或者数据压缩成摘要，使得数据量变小

  • MD5不是加密算法，因为它是单向加密。
  • 在加密的时候加上额外内容

HTTP、HTTPS的异同

https使用tls技术，支持安全传输

TLS： 传输层安全协议

  1. 客户端生成随机数1，将随机数、协议版本、加密算法传给服务端
  2. 服务端生成随机数2，确认加密算法，把数字证书和随机数2传给客户端
  3. 客户端确认数字证书是否有效
  4. 客户端生成随机数3，并用服务端公钥加密，并传输给服务端
  5. 双方根据随机数1、2、3和加密算法生成对称密钥
  6. 使用对称密钥进行加密通信

不直接用非对称加密的原因：非对称加解密成本高

DNS 的工作原理

tcp 和 udp 的区别

  • tcp 提供可靠有连接服务。udp 提供不可靠无连接服务
  • tcp 保证按序到达。udp 只管发送，什么都不保证
  • 因此udp 比tcp 更快，占用更少性能

TCP

  • 序号（seq）：4个字节。表示报文的第一个字节的序号
  • 确认号（ack）：4个字节。期待收到对方下一个的第一个字节（表示之前的都已经收到了）
  • 控制位：6个字节。URG紧急位\ACK确认位\PSH推送位\RST重置位\SYN同步位\FIN终止位
  • 窗口：2字节。指明允许对方发送的数据量

TCP实现可靠有连接传输

  • 确认和重传机制。发送方将数据分为多个TCP报文段，并分配一个序列号。接收方收到后发送确认应答。如果一定时间未收到，就重传。
  • 滑动窗口。发送方和接收方都维护一个滑动窗口，用于控制发送和接收的数据量。接受方确认序号后，发送方的窗口滑到已确认号+1。滑动窗口可以有效控制流量和防止数据丢失
  • 流量控制。基于接收方的滑动窗口进行流量控制
  • 拥塞控制。慢开始（初始窗口小，成倍扩大，指数级增长）、拥塞避免（发送方达到阈值后，窗口以线性增长）、快速重传（3个重复确认应答后，认定拥塞，发送方重传丢失报文而不是等待超时）、快速恢复（快速重传后，发送方将拥塞窗口减半，然后没收到一个确认应答，窗口就增加一个报文段大小）
  • 数据校验。双方同时计算校验和，不匹配重传

如何实现可靠的udp

  • 超时重传
  • 接收方确认
  • 消息的唯一序号和重传窗口
  • 错误检测和纠正

TCP 四次挥手

time-wait

  • time-wait 状态，主动方没有完全关闭tcp连接，端口处于不可用
  • 2MSL，确保被动端能接收到确认关闭报文；确保所有连接中的报文已过期

tcp粘包

应用层没有对tcp的传输内容进行区分，导致多段内容拼在一起

解决：

  1. 应用层确认长度
  2. 应用层定义边界字符

VPN 原理

Virtual Private Network。在公共网络上建立加密连接技术。能在不安全的公共网络上创建一个安全通道，保护用户隐私和安全性。
