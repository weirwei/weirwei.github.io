使用遗传算法实现智能排课系统

背景

  1. 排课流程复杂：课程、课班（人数、性别、年级、节次）、老师（学科、时间）、教室（容量、类型、时间）、时间（日期、节次）、教具
  2. 凸显竞争力

基础设计

约束设计：

  • 硬约束条件：必须满足的条件，如果不能满足，则认为课程无法排班。
  • 软约束条件：尽量满足的条件，满足的软约束越多，表示排课的结果越好。
  • 静态约束：约束条件不会跟随已经排的课程的变化而变化，也即可用在排课前就能够确认时间、地点、教室是否可用。
  • 动态约束：约束条件会跟随已经排的课程而变化，也即不可在排课前就确定时间、地点、教室是否可用。

对象：

  • 课班
  • 老师
  • 教室
  • 时间

整体流程

  1. 在数据库记一个任务

难点

  1. 性能优化（运行时间、内存瓶颈）
    • 实现多 pod 负载均衡
    • 并发控制
    • 简化类型 int -> uint8
    • 采用内存池的方式
  2. 排课效果
    • 交叉效果优化
      • 编写一个效果分析工具，可以输出交叉比例效果图
      • 修改交叉率
    • 打分函数优化
      • 结合实际效果分析

func Hungarian(itemList []Item) []Matcher {
	m := fmtItemList(itemList)

	// 反向记录
	rev := make(map[int]int)
	for idx := 0; idx < len(m); idx++ {
		vis := make(map[int]bool)
		if !match(m, rev, vis, idx) {
			return nil
		}
	}
	var res []Matcher
	for rmtId, sIdx := range rev {
		res = append(res, Matcher{
			Source: itemList[sIdx].Source,
			Remote:  rmtId,
		})
	}
	sort.Slice(res, func(i, j int) bool {
		return res[i].Source < res[j].Source
	})
	return res
}

func match(m [][]int, rev map[int]int, vis map[int]bool, idx int) bool {
	opts := m[idx]
	for i := 0; i < len(opts); i++ {
		rmtId := opts[i]
		if _, ok := vis[rmtId]; ok {
			continue
		}
		vis[rmtId] = true
		if lastIdx, ok := rev[rmtId]; !ok || match(m, rev, vis, lastIdx) {
			rev[rmtId] = idx
			return true
		}
	}
	return false
}

func fmtItemList(itemList []Item) [][]int {
	m := make([][]int, len(itemList))
	for i, item := range itemList {
		m[i] = item.Optional
	}
	return m
}

商品列表性能优化

背景

系统：

  • 小程序服务端：小程序直接对接的服务端，提供商品列表、下单、个人中心、订单中心等用户功能
  • 售卖系统：对接售卖中台的中间系统，主要有记录商品sku、订单等功能，是我们业务对售卖中台的一个中转服务。同时也实现了偏向业务的售卖功能
  • 课程系统：维护教学相关的实体模型（学校、学生、老师、教师、课程等）。围绕这些实体模型为上层业务提供支持

获取商品列表慢

  • 查商品列表。需要uid
    • 获取学校下所有上架的sku
    • 过一遍策略（uid）（可以限制哪些学生可以买）
    • 获取渠道库存
  • 查课班信息。需要schoolId
  • 查询教具商品信息。
  • 查用户的报名信息。需要uid

课程系统查课程详情

  1. 查询课班信息
  2. 根据课班信息查询课程资源
  3. 根据课班id查询章节信息
  4. 查询教具关联表查询教具
  5. 过课班的策略

分析性能问题

  1. 观察监控，发现瓶颈在课程系统的数据库
  2. pprof 分析课程系统接口性能

方案

  1. 把策略迁移到售卖系统
  2. 对课程系统查询到的课班列表做缓存

结果

QPS 从 600 -> 800
