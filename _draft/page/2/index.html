init 函数

  • init 函数在main 函数执行之前系统自行调用
  • init 函数不能被调用
  • 不同包下的init 函数没有特定的执行顺序
  • import _ [第三方包] 会执行这个包的init 函数

数组和切片

  • 数组传递是值拷贝，因此建议用切片
  • 切片append 超过原切片的cap，就会重新分配底层数组
  • go切片扩容规则：
    • 如果切片的长度小于1024，则扩容后的容量会变为原来的2倍。
    • 如果切片的长度大于等于1024，则扩容后的容量会变为原来的1.25倍。
  • 切片copy 是将切片的值赋值到另一个切片，允许共用同一个底层数组

如何识别垃圾

引用计数法

在对象上记录自己被引用的次数，被引用时+1，引用对象被回收时-1，引用数为0时标记为可回收对象

优点：

  • 简单，回收快

缺点：

  • 需要额外的空间存放计数
  • 无法处理循环引用
  • 频繁更新计数，降低性能

追踪式回收算法

从GC roots中标记所有变量，不可达的标记为垃圾

聚簇索引 和非聚簇索引

  • 聚簇索引：一般是根据主键创建的，会按照索引列的顺序进行物理上的重新排序，使数据相邻存储，加快数据的读取，减少随机访问
  • 非聚簇索引：普通索引，提高查找效率
