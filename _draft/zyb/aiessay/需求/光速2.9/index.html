V2.9【跟版】柯欣&常锐：议论文讲解模式

背景&资料

https://wiki.zuoyebang.cc/pages/viewpage.action?pageId=502362895

功能概要

成文流程：

  1. 提取题目：图片过ocr。矫正。上传cos
  2. 审题：算法根据题目直接返回 关键句和审题prompt
  3. 请求大模型获取关键句和审题信息。
  4. 图片划关键句。
  5. 立意：算法提供立意离线数据或者返回prompt，aiwriting 请求大模型获取信息。前置：标题。内容可以编辑
  6. 框架构思：算法返回框架离线数据或者返回prompt，aiwriting 请求大模型获取信息。前置：标题+立意。内容可以编辑
  7. 素材：多选，算法返回素材离线数据或者返回prompt，aiwriting 请求大模型获取信息。前置：标题+立意+框架。内容可以编辑
  8. 成文：算法返回成文prompt，aiwriting 请求大模型生成文章，展示前面4个阶段的所有信息，并流式生成文章
    • 任意一步都可以直接成文，此时前四个阶段未生成的内容也流式生成
  9. 重写一篇：
    • 换立意：支持更换立意。点击更换意图后，直接跳转成文流程选择立意的步骤
    • 换框架：支持更换框架。点击换框架后，直接跳转成文流程选择框架构思的步骤
    • 换素材：支持更换框架。点击换框架后，直接跳转成文流程选择素材的步骤

寸功相关：

  1. 同步：
    • 没过寸功的内容不展示，如果没有内容展示，提示刷新，刷新后再次请求，如果还是没有，返回首页
  2. 异步：
    • 生成过程中被撤回：撤回对应的内容。点击下一步的时候，检查已选择内容寸功状态，如果发现有选中的内容被撤回，则回到审题页

确认：

  1. 是否存在多个前置内容生成一个后置内容的情况
  2. 寸功异步撤回，成文页的内容撤回策略是否同文章撤回策略

设计方案

重点问题：

  1. 存储：数据存储做在哪儿？我们记录全量数据，还是算法也承接一部分
  2. 历史记录是什么时候生成的，需要记录生成中的状态吗？成文之后生成
  3. 直接成文立意、框架等，生成几个
  4. 立意、框架、素材 可以编辑，新增还是覆盖
  5. 图片关键句高亮怎么搞？

后处理机制：两段重复、同样的文本一直输出

模型设计：

  1. 生产素材
  2. 流程数据
    • 阶段信息：记录有哪些阶段和当前阶段
    • 阶段数据：每个阶段需要记录的数据
      • 审题：中间数据：prompt、关键句、题目。
      • 立意：中间数据：
      • 框架构思
      • 素材选取

流程设计：

  1. 拍照识别
  2. 创建「写作流程」，关联识别信息
  3. 生成「审题」，记录内容（ActionRecord & ArticleParagraph）
  4. 「写作流程」关联「审题」
  5. 生成「立意」，记录内容（ActionRecord & ArticleParagraph）
  6. 选择「立意」，「写作流程」关联「立意」
  7. 生成「框架构思」，记录内容（ActionRecord & ArticleParagraph）
  8. 选择「框架构思」，「写作流程」关联「框架构思」
  9. 生成「素材选取」，记录内容（ActionRecord & ArticleParagraph）
  10. 选择「素材选取」，「写作流程」关联「素材选取」
  11. 「AI成文」，生成文章，关联写作流程
  12. 「按要求重写」

sequenceDiagram
    participant client as 客户端
    participant FE as 前端
    participant server  as ai-writing
    participant ag as 算法
    participant llm as LLM
    
    par 获取高考题
       FE ->>+ server: 写作页面获取高考题「年份（可选）」
       server ->> server: 查DB
       server ->> server: 根据用户 ip 和当前时间推荐题目
       server -->>- FE: 高考题信息
       
    and 拍题
    client ->>+ server: 上传图片
    server -->>- client: succ
    
    client ->>+ server: 识别图片
       server ->>+ ag: ocr「图片url」
       ag -->>- server: 矫正后的图片base64
       server ->> server: 写数据库「tblWritingPicRecognize」
    server -->>- client: 识图结果
    client ->>+ FE: 识别id

   and 审题
      FE ->>+ server: 获取审题信息「ocrId 或 题id」
         server ->> server: 获取题目内容
         
         server ->>+ ag: 获取审题prompt「题目内容」
         ag -->>- server: 关键句、离线的审题内容或审题prompt
         
         opt 来源是图片
         server ->>+ ag: 根据关键句获取图片坐标
         ag -->>- server: 图片坐标
         end
      server -->> FE: 关键句信息「关键句内容、坐标」
         opt 算法返回的是审题 prompt
         server ->>+ llm: 生成审题内容「prompt、题目内容」
         llm -->>- server: 审题内容
         end
         server ->> server: 写数据库「tblActionRecord, tblArticleParagraph」
      server -->>- FE: 审题内容、sessionId
   
   and 立意、框架构思、素材选取、AI成文、重写一篇、再生成一个
       FE ->>+ server: 获取流程信息「当前阶段、目标阶段、当前阶段信息、sessionId、preSid(重写一篇)、describe(重写一篇)」
          note left of FE: 阶段信息：<br>- 前置内容id <br>- 生成内容数量 <br>- 已存在的内容id(再生成一条)
          par 立意
          note right of FE: 跨阶段执行时，会顺序执行 <br>「当前阶段」至「目标阶段」之间的流程 <br>默认选中返回结果第一条
          
          server ->> server: 根据审题id查询已存在的立意（不包括已存在的ids）
          server ->> FE: 立意内容
          opt 已存在立意不足需求数量
             server ->>+ ag: 获取立意prompt「题目内容、审题信息、已生成立意」
             ag -->>- server: 离线立意 或 立意prompt
             
                opt 算法返回的是 prompt
                server ->>+ llm: 生成立意内容「prompt」
                llm -->>- server: 立意内容
                end
             server ->> server: 写数据库「tblActionRecord, tblArticleParagraph」
             server -->> FE: 立意内容
          end
          
          and 框架构思
            server ->> server: 根据立意id查询已存在的框架（不包括已存在的ids）
            server ->> FE: 框架内容
               opt 已存在框架不足需求数量
               server ->>+ ag: 获取框架构思prompt「题目内容、审题信息、立意、已生成框架构思」
               ag -->>- server: 离线框架构思 或 框架构思prompt
               
                  opt 算法返回的是 prompt
                  server ->>+ llm: 生成框架构思内容「prompt」
                  llm -->>- server: 框架构思内容
                  end
               server ->> server: 写数据库「tblActionRecord, tblArticleParagraph」
               server -->> FE: 框架构思内容
               end
          
          and 素材选取
            server ->> server: 根据框架id查询已存在的素材（不包括已存在的ids）
            server ->> FE: 素材内容
            opt 已存在框架不足需求数量
             server ->>+ ag: 获取素材选取「题目内容、审题信息、立意、框架构思、已生成素材选取」
             ag -->>- server: 素材
             server ->> server: 写数据库「tblActionRecord, tblArticleParagraph」
             server ->> FE: 素材
           end  
          and AI成文
          opt 重写一篇（preSid 不为空）
          server ->> server: 查询 preSid 的文章
          end
          server ->>+ ag: AI成文「题目内容、审题信息、立意、框架构思、素材、上一篇文章(重写一篇)、describe(重写一篇)」
          ag -->>- server: 离线文章 或 成文prompt
             opt 算法返回的是 prompt
             server ->>+ llm: 生成文章内容「prompt」
             llm -->>- server: 成文内容
             end
          server ->> server: 写数据库「tblArticleResult, tblArticleParagraph, tblSessionRecord」
          server -->> FE: 成文内容
          end
       
       server -->>- FE: close
    end
